API Debug Tool 质量保障平台 - Phase 2 详细设计文档
======================================

一、Phase 2 概述
------------

### 1.1 阶段定位

在 Phase 1 建立的数据持久化与基础用例管理能力之上，Phase 2 重点构建**自动化测试能力**，实现从手动调试到自动化回归的升级。

### 1.2 核心目标

| 目标          | 说明                      | 成功标准                  |
| ----------- | ----------------------- | --------------------- |
| 数据驱动测试      | 支持 CSV/JSON 数据源，参数化执行用例 | 单个用例可绑定数据集，遍历执行       |
| 定时任务系统      | 支持 Cron 表达式配置定时执行       | 任务按时触发，执行记录可追溯        |
| 测试报告增强      | 多维度报告与趋势分析              | 生成 HTML/PDF 报告，展示趋势图表 |
| 通知告警        | 执行结果多渠道通知               | 支持邮件、钉钉、飞书告警          |
| 压力测试        | 并发请求与性能指标采集             | 支持并发设置，展示 TPS/RT 曲线   |
| Mock Server | 根据用例自动生成 Mock 接口        | 支持动态响应规则配置            |

### 1.3 与 Phase 1 的关系

text

Phase 1 产出                      Phase 2 扩展
─────────────────────────────────────────────────
用例管理 ──────────────────────► 数据驱动测试
环境管理 ──────────────────────► 定时任务配置
场景编排 ──────────────────────► 场景模板化
执行记录 ──────────────────────► 报告生成引擎
CLI 工具 ──────────────────────► 定时任务触发器

### 1.4 不包含内容

* 单元测试与覆盖率集成（Phase 3）

* 缺陷管理模块（Phase 3）

* 多用户与权限（Phase 4）

* 智能分析（Phase 4）

四、核心引擎设计
--------

### 4.1 数据驱动引擎

**设计目标**：支持 CSV/JSON 数据集，实现用例的参数化执行

**数据源类型**：

| 类型   | 格式示例              | 使用场景       |
| ---- | ----------------- | ---------- |
| CSV  | 第一行为列名，后续行为数据     | 大量数据、表格数据  |
| JSON | 数组格式，每个元素为一个对象    | 结构化数据、嵌套数据 |
| 内置函数 | 随机数、时间戳、UUID 等生成器 | 动态数据生成     |

**执行模式**：

| 模式   | 说明        | 适用场景   |
| ---- | --------- | ------ |
| 串行执行 | 逐行遍历数据集   | 普通功能测试 |
| 失败停止 | 遇到失败即停止   | 核心链路验证 |
| 失败继续 | 记录失败但继续执行 | 数据清洗验证 |

**变量引用语法**：

text

# CSV 数据行变量

{{ dataset.user_id }}
{{ dataset.email }}

# 内置函数变量

{{ $random.int(1, 100) }}
{{ $timestamp.now() }}
{{ $uuid.v4() }}
{{ $random.string(10) }}
{{ $random.email() }}

### 4.3 压力测试引擎

**设计目标**：支持并发请求，实时采集性能指标

**核心参数**：

| 参数   | 说明            | 范围        |
| ---- | ------------- | --------- |
| 并发数  | 同时执行的请求数      | 1-1000    |
| 总请求数 | 目标请求总数        | 1-1000000 |
| 持续时间 | 压测持续时间（秒）     | 1-3600    |
| 预热时间 | 逐步增加并发的时间     | 0-300     |
| 阶梯加压 | 按时间/请求数自动增加并发 | 配置项       |

**压测模式**：

| 模式    | 说明           |
| ----- | ------------ |
| 固定并发  | 保持恒定并发数      |
| 阶梯加压  | 按配置逐步增加并发    |
| 峰值冲击  | 短时间内高并发，然后回落 |
| 稳定性测试 | 长时间固定并发      |

**采集指标**：

| 指标              | 说明      | 单位  |
| --------------- | ------- | --- |
| TPS             | 每秒事务数   | 次/秒 |
| RT              | 平均响应时间  | ms  |
| P50/P90/P95/P99 | 响应时间分位数 | ms  |
| 成功率             | 请求成功率   | %   |
| 错误数             | 失败请求数   | 次   |
| 并发连接数           | 实时并发数   | 个   |

### 4.4 Mock 引擎

**设计目标**：根据规则自动生成 Mock 接口，支持动态响应

**Mock 规则匹配流程**：

text

┌─────────────────────────────────────────────────────────────┐
│                    Mock 请求匹配流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  收到请求 → 匹配路径和方法 → 匹配条件校验 → 生成响应 → 返回    │
│                                                             │
│  匹配优先级：                                                │
│  1. 精确路径匹配                                            │
