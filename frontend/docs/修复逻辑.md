**整体目标**

修复用例模块分类体系和弹窗下拉问题，同时统一深色/浅色主题下的下拉样式：

1. “按创建时间”“降序”两个筛选下拉在深色主题下背景不匹配
2. 新增分类的“上级分类”下拉被遮挡或无数据
3. 新建用例的“所属分类”下拉被遮挡或无数据
4. 增加系统默认“根目录”
5. 未指定分类的用例归入“根目录”
6. 当前所有顶级目录移动到“根目录”下面
7. “根目录”作为系统目录，不允许编辑/删除
8. 修复 Cannot access 'rootFolder' before initialization

**前端样式修复**

先处理 Element Plus 下拉弹层的样式问题。因为 el-select / el-cascader 的弹层会 teleport 到 body，组件 scoped 样式经常覆盖不到，所以要给弹层指定全局 popper-class。

排序筛选下拉：

```
<el-select  ...  popper-class="case-sort-select-popper" /> 
```

然后在全局样式文件中写：

```
.case-sort-select-popper {  background: var(--bg-panel);  border: 1px solid var(--border-default);  box-shadow: 0 20px 58px rgba(0, 0, 0, 0.48); } .case-sort-select-popper .el-select-dropdown__item {  color: var(--text-primary); } .case-sort-select-popper .el-select-dropdown__item:hover, .case-sort-select-popper .el-select-dropdown__item.is-selected {  background: rgba(0, 255, 255, 0.1);  color: var(--neon-cyan); } 
```

浅色主题再用 .theme-light .case-sort-select-popper 覆盖背景、边框、文字色。

**分类级联下拉修复**

新增分类和新建用例都要用同一个 cascader popper class：

```
<el-cascader  ...  append-to-body  popper-class="folder-cascader-popper" /> 
```

全局样式里维护：

```
.folder-cascader-popper.el-popper, .folder-cascader-popper.el-cascader__dropdown {  z-index: 100200 !important;  background: var(--bg-panel);  border: 1px solid var(--border-default);  border-radius: 8px;  box-shadow: 0 20px 58px rgba(0, 0, 0, 0.48); } 
```

同时覆盖 .el-cascader-panel、.el-cascader-menu、.el-scrollbar、.el-cascader-node 的背景和文字色。这样可以同时解决“被遮挡”和“深色主题下白底”的问题。

**后端根目录逻辑**

在 case_folders.py 里定义系统根目录：

```
ROOT_FOLDER_NAME = "根目录" ROOT_FOLDER_PATH = "/根目录" 
```

增加统一准备函数：

```
def prepare_root_folder(db, tenant_id):    root = ensure_root_folder(db, tenant_id)    normalize_root_children(db, tenant_id, root)    normalize_uncategorized_cases(db, tenant_id)    return root 
```

职责拆分：

- ensure_root_folder：不存在则创建“根目录”
- normalize_root_children：把所有 parent_id is None 且不是根目录的分类移动到根目录下面
- normalize_uncategorized_cases：把 folder_path 为空、""、"/" 的用例移动到 /根目录

所有分类查询入口都先执行：

```
prepare_root_folder(db, tenant_id) 
```

包括：

- 分类树接口
- 分类平铺接口
- 确保根目录接口

**系统目录保护**

后端返回分类时增加：

```
is_fixed: bool 
```

判断逻辑：

```
def is_system_folder(folder):    return folder.name == ROOT_FOLDER_NAME and folder.parent_id is None 
```

更新和删除接口中直接拦截：

```
if is_system_folder(folder):    raise HTTPException(status_code=400, detail="系统目录无法编辑") 
```

删除同理返回“系统目录无法删除”。

**创建分类逻辑**

新增分类时，如果前端没有传 parent_id，后端默认挂到根目录：

```
parent_id = data.parent_id or root.id 
```

同时禁止创建同名系统根目录：

```
if data.name == ROOT_FOLDER_NAME:    raise HTTPException(status_code=400, detail="根目录为系统目录") 
```

**用例分类默认值**

在 cases.py 里增加统一归一化：

```
def normalize_folder_path(value: str | None) -> str:    if not value or value == "/":        return "/根目录"    return value 
```

创建和更新用例时都走这个函数，避免新数据再次落到空分类。

**新增分类弹窗**

CaseFolderDialog.vue 中增加“上级分类”字段：

```
<el-cascader  v-model="form.parent_id"  :options="folderOptions"  :props="cascaderProps"  append-to-body  popper-class="folder-cascader-popper" />

打开弹窗时加载分类树：

watch(visible, async (opened) => {  if (!opened) return  await loadFolders()  if (!isEdit.value && form.value.parent_id == null) {    form.value.parent_id = rootFolderId.value  } }) 
```

编辑系统目录时禁用输入并阻止提交：

```
if (form.value.is_fixed) {  ElMessage.warning('系统目录无法编辑')  return } 
```

**分类管理列表**

CaseFolderManager.vue 初始化时先调用：

```
await ensureRootFolder() await loadFolders() 
```

列表中展示 is_fixed，系统目录显示标识，隐藏或禁用编辑/删除按钮。

**新建用例弹窗**

CaseManagement.vue 中把原来的手输 folder_path 改为 cascader：

```
<el-cascader  v-model="form.folder_id"  :options="folderOptionsWithRoot"  :props="{ checkStrictly: true, label: 'label', value: 'id', emitPath: false }"  append-to-body  popper-class="folder-cascader-popper" /> 
```

保存时由 folder_id 找到分类并转换为 folder_path：

```
const folder = getFolderById(submitData.folder_id) || rootFolder.value submitData.folder_path = folder ? '/' + folder.name : '/根目录' 
```

编辑用例时，把已有 folder_path 反查成 folder_id：

```
form.value.folder_id = getFolderIdByPath(c.folder_path) ?? rootFolder.value?.id ?? null 
```

**TDZ 报错修复**

不要在 defaultForm() 中访问 rootFolder，因为 form = ref(defaultForm()) 执行时 rootFolder 还没初始化。

错误写法：

```
folder_id: rootFolder.value?.id ?? null 
```

修复为：

```
folder_id: null 
```

然后在分类加载完成后再设置默认根目录：

```
if (!editingCase.value && form.value.folder_id == null) {  form.value.folder_id = rootFolder.value?.id ?? null } 
```
